Welcome to ComposeStar StarLight

Compose* (or ComposeStar) is a project that aims at enhancing the power of component- and object-based programming, so that software becomes easier to structure and modularize, hence easier to develop, maintain and extend.

In particular, ComposeStar offers aspect-oriented programming (see AOSD) through the composition filters model (see project site).

StarLight is an implementation of ComposeStar specific for the Microsoft .NET Framework version 2.0. Filter code is injected into your existing assemblies based on the specifications and selections in the concern files.

For more information about ComposeStar, the concern language, and more details about the Composition Filters approach, have a look at the ComposeStar website.

Filters and Weaving

StarLight has a number of buildin filter actions, filter types and weaving strategies. Besides these standard items, it is also possible to create your own filter types , filter actions , and weaving strategies .

Take a look at the filters page or the weaving page for more information or look for the specific types in the class help.

Visual Studio Integration

StarLight integrates into Microsoft Visual Studio 2005 and offers a project system to create your own *.*cpsproj files and a language service to enable syntax highlighting and IntelliSense for ComposeStar Concern (*.cps) files. See the Visual Studio Integration pages for more information.

MSBuild

The StarLight weaving process consists of a number of MSBuild Tasks which are executed from the StarLight target file in the MSBuild folder. These tasks are responsible for analyzing the source files, determining the location of the filters, and the actual weaving of the new instructions. For more information about the MSBuild process, visit the help pages about MSBuild.
