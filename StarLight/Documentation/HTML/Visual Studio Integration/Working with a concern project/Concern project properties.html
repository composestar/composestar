Concern project properties

The concern project properties are divided into general and build property groups. The general settings apply to the complete project, while the build settings are only used in the specified configuration. For example, you can run the IL Verifier in Debug mode, and disable the verifier in release mode.

General project properties

Setting Description
Assembly Name
Name of the assembly to be created if the concern project contains CSharp, JSharp, or VB.NET code.
Default Namespace
The default namespace used for newly created items.
Output Type
Specifies the output type of the concern project if it contains code. Select from Console Application, Class Library, Windows Application.
Repository filename
The name of the xml file the StarLight compilation information is stored in.

Build project properties

Setting Description
Debug level
The debug level of the StarLight Java part. Can be one of the following choices; Error, Crucial, Warning, Information, Debug.
Verify IL
True if the assemblies should be verified using the PeVerify tool (SDK required) after weaving, false otherwise
Weave Debug Level
Specifies the debug level of the weaver. Set it to None to disable debugging. The Statistics settings shows the number of weaved filters, internals and externals, and some timing information. The Detailed setting logs all the injected IL code and has extensive timing information. The results are store in the output folder in the {assemblyname}.weavelog.txt and {assemblyname}.weavetiming.txt files. The timing file can, for instance, be imported into Microsoft Excel so you can plot the amount of time each weave actions takes.

You can also change the WeaveDebugLevel for a single run by specifying the property. For instance: 
MSBuild Tracing.cpsproj /p:WeaveDebugLevel=Detailed
