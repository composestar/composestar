Debugging of woven files

The StarLight IL weaver updates existing debug files (PDB) by rewriting the sequence points and documents from the original debug file and hides the injected code. Debugging the woven code is no different then debugging your original program because the extra added filter code is hidden for the debugger.

The woven file and new PDB file are located in the bin folder of your concerns project. There are a number of ways to use debugging:

  • Copy the woven assembly and PDB file to the output folder of your original project. You can do this manually or use a postbuild target like the AfterWeave. In this MSBuild target, use a copy task to copy the woven files and PDB back to your project folder.
  • In the properties of your source project, select the Debug tab and select the option Start external program. Browse for the executable in the concern project output folder.
  • Attach the Visual Studio debugger to an instance of the application. Start the woven application, select Debug,Attach to Process... from the menu and pick your running application from the list.
  • Use the CORDBG located in the .NET SDK folder or the DbgCLR for a GUI version of the debugger.

Note: the debugger will skip filter code, meaning a number of actions can occur which cannot be seen in the debugger.
