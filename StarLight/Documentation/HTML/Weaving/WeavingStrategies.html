Weaving Strategies

The weaving model determines, at a high level, what type of code has to be generated. The IL weaver is reponsible for generation the correct IL statements like creating branches, jumps, conditions, calls, and so on. A visitor design pattern is used to accomplish this.

Filter Action code however is handled by dedicated weaving strategies. The default Filter Actions have each a buildin weaving strategy which determines what type of IL code has to be generated.
These buildin weaving strategies can be found in the StarLight installation folder/WeaveStrategies folder.

If there is no weaving strategy for a Filter Action, the Execute method will be used.

If you have created your own Filter Action and do not want to use the Execute method, you can create your own weaving strategy.
