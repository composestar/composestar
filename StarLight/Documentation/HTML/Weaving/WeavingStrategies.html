Weaving Strategies

The weaving model determines, at a high level, what type of code has to be generated. The IL weaver is reponsible for generating the correct IL statements, like creating branches, jumps, conditions, calls, and so on. A visitor design pattern is used to accomplish this task.

Filter Action code however is handled by dedicated weaving strategies. The default Filter Actions have each a builtin weaving strategy which determines what type of IL code has to be generated.
These builtin weaving strategies can be found in the StarLight installation folder/WeaveStrategies folder.

If there is no weaving strategy for a filter action, the Execute method of the filter action will be used. If you have created your own filter action and do not want to use the Execute method, you can create your own weaving strategy. Since it is harder to create IL code than C# or VB.NET code, only use this method when you have knowledge about IL and can create more optimized solutions by injecting direct IL.
