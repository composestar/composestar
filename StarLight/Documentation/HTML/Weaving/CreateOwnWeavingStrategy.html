Create your own weaving strategy

If you have created your own Filter Action and do not want to use the Execute method of the Filter Action, then create your own weaving strategy. This gives you direct control over the emitted IL code, but introduces a more complex model to inject code into assemblies.

How to create a weave strategy

A custom weave strategy is basically a type implementing the FilterActionWeaveStrategy . Besides the inheritance, you also have to add a WeaveStrategyAttribute to the class so it can be discovered and added to the weaving strategies by the IL Weaver.

The next step is to add your weave strategy to the list of strategies used by the weaver. Place your assembly containing the weave strategy implementation in the StarLight installation folder/WeaveStrategies folder. When the IL weaver is started for the first time, it will read all the assemblies in this folder, looks for the WeaveStrategyAttribute, and add the found strategies to the list.

To help you in your development of a custom weave strategy, you can use the WeaveStrategyUtilities , the CecilUtilities , and JoinPointContext classes during the development.

The WeaveStrategyUtilities contains functionality to work with arguments and Join Points, the CecilUtilities has functions to resolve methods and types from assemblies so they can be used in your code.
