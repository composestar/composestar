MSBuild

The StarLight project uses the Microsoft Build system to perform the analysis and weaving of the concern code. The cpsproj file for a StarLight Concern project consists of a number of tasks controlling the compilation and weaving. You can start MSBuild by either running MSBuild (located in your .NET framework folder) with the name of the cpsproj file, or use the Build option in Visual Studio on the Concern project.

For instance, to weave a Concern project named Tracing.cpsproj, execute the following command from the command line:

MSBuild Tracing.cpsproj

A special Weave target is being called as the default target. To clean and weave the project, use the following construction:

MSBuild Tracing.cpsproj /t:reweave

The cpsproj file contains an import of the StarLight.targets file, located in the MSBuild folder under the Program Files folder. This target file contains the tasks definitions and controls the compilation and weaving. It will build the referenced projects first, copy the output to the bin folder of the concern project, and starts the analyzer and weaver.

For more information about the tasks, see the StarLight Tasks page. To learn how to extend the MSBuild process, see the extending the MSBuild section.
