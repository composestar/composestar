Build-in Filter Types

The following build-in filter types are available:

Filter Name Description
After
Executes an AdviceAction on return (after dispatch).
Before
Executes an AdviceAction on call (before dispatch).
Dispatch
Executes a DispatchAction.
Error
Executes an ErrorAction.
Substitution
Executes a SubstitutionAction.

Defining new Filter Types

A new filtertype can be defined by creating a class extending the abstract class FilterType . No method needs to be implemented for a filter type, although it should contain a custom attribute FilterTypeAttribute , containing the following information:

  • The unique name of the filter type.
  • The names of the four different filter actions.

Note The filter type class is only used at compiletime.

Examples:

  • EmptyFilter
  • TracingFilter
  • ProfilingFilter
