Conditional Superimposition

The superimposition of filter modules can be made conditional. At runtime the condition will be evaluated to decide whether to execute the filter module.

Specifying the Conditions

To make a superimposition conditional, first a condition needs to be specified in the 'superimposition' block. This can be done by using the 'conditions' tag. Below the 'conditions' tag, the conditions are specified with the following layout:

<id> : <namespace>.<classname>.<methodname>;
<id> gives and id to the condition.
<namespace>.<classname>.<methodname> identifies the method used to obtain the truth value of the condition at runtime.

Multiple options are available for the parameters of the condition method. How to create a condition method is explained in Creating Condition Methods for Conditional Superimposition.

superimposition

{

    conditions

            conditionId : NameSpace.ClassName.MethodName;

    selectors

        ...

    filtermodules

        ...

Specifying Conditional Superimposition

The conditional superimposition can now be specified by placing the condition-id, followed by '=>', before a superimposition specification:

superimposition

{

    conditions

            ...

    selectors

        ...

    filtermodules

        conditionId => someSelector <- SomeFilterModule;

}

Example

The following listing shows an example of how conditional superimposition can be used for a logging filter module:

concern LoggingConcern in LoggingModule

{

    filtermodule LoggingFM

   {

            inputfilters

            log: Logging = { True => [*.*] }

   }

   

   

    superimposition

    {

        conditions

                  enabled : LoggingModule.Logging.IsEnabled;

            selectors

            logClasses = ...

        filtermodules

            enabled => logClasses <- LoggingFM;

    }

}
